# 薙刀式 v17ベスト版 keymapper 設定ファイル

本リポジトリは、大岡俊彦 氏考案の日本語入力配列**「薙刀式（なぎなたしき）v17ベスト版」**（2026年1月発表）を、クロスプラットフォーム対応のキーリマップソフト `houmain/keymapper` で実現するための設定ファイルです。

Windows、MacOS、Linux の3つのOSに対応し、同一のタイピング環境を提供します。

## 概要

キーボードの「単打」と「同時押し」を高度に組み合わせることで、指の移動を最小限に抑え、高速かつ快適な日本語入力を可能にします。

### 主な機能
* **3つのレイヤー構造:** かな入力（通常）、Lower（数字・カーソル移動・編集）、Raise（記号入力）、Adjust（音量・メディア・Fキー）。
* **強力な編集モード:** 左手キー（`D F` または `C V`）との同時押しにより、右手を即座にカーソル移動・選択・コピペが可能な「編集用コントローラー」に変形。
* **シフト英数モードからの自動復帰:** アルファベット大文字入力で一時的に英数モードへ移行し、`Enter` を押すだけで自動的にかなモードに復帰。
* **クリップボード経由の高速文字入力:** 小説執筆などで多用する記号（`……`、`――`）や登録された固有名詞を、OSの仕様差を超えて高速かつ確実に入力。

## 導入手順（OS別準備）

本設定ファイル（`keymapper`）の他に、OSごとにクリップボード管理やIME制御のための補助ツールが必要です。

### 1. 全OS共通の準備
1. [houmain/keymapper](https://github.com/houmain/keymapper) (Version 5.5.0以上) をダウンロードします。
2. ご自身のOSおよびCPUアーキテクチャ（Intel / Apple Silicon / AMD等）に適合するバイナリをインストールしてください。

### 2. OS別の個別セットアップ

#### Windows
* **必要な追加ソフト:** なし
* **備考:** 標準のIME機能およびキー送信のみで動作するため、事前準備は不要です。

#### MacOS
* **必要な追加ソフト:** `copyQ` (GUI版)
* **導入手順:**
  1. `copyQ` をインストールします。
  2. コマンドライン（CLI）から `copyq` コマンドを直接実行できるよう、環境変数（パス）を通すか、シンボリックリンクを設定してください（※詳細は copyQ の公式ドキュメントを参照）。
  3. 標準搭載の `pbcopy` と連携して動作します。
* **その他の設定:**
Mac側で「Windows風のキー操作」を有効にしておく必要があります。
設定手順は以下のとおりです。
  1. システム設定 を開く
  2. キーボード を選択
  3. 日本語入力 を開く
  4. 詳細 をクリック
  5. Windows風のキー操作 を ON にする

#### Linux
* **必要な追加ソフト:** `xsel` (または環境に応じて `xclip`, `wl-clipboard`)
* **導入手順:**
  1. パッケージマネージャから `xsel` をインストールします（例: `sudo apt install xsel`）。
  2. 独自のクリップボード（セカンダリバッファ）を利用して動作します。
* **その他の設定:**
Mozc のキー設定で以下の割り当てが行われている必要があります。
Mozc の設定画面から「キー設定」を開き、下記の割り当てを設定してください。
  1. 直接入力　→　Henkan　：IME を有効化
  2. 入力文字なし → Muhenkan：IME を無効化

## keymapperに設定ファイルを読み込ませる
* トレイアイコン メニューからConfigurationを選択
* Configurationファイル(keymapper.conf)が開きます。
* naginata_v17_copyq.confのテキストをコピーして保存します
* トレイアイコンメニューからReloadで更新します。
* hj 同時押しで 薙刀式の入力開始、IMEが日本語入力（ローマ字入力モード）に変わります
* その状態でJキー単打で『あ』と入力できれば設定完了です
* fg 同時押しで 薙刀式の英数モードで入力できます

## システム内部設計（OSごとの文字入力方式）

記号や固有名詞を高精度かつ高速に入力するため、OSごとに最適化された異なるアプローチを内部で自動的に切り替えています。

### Windows：直接入力方式
一時的にIMEの入力を完了（確定）させ、対象文字列をダイレクトに流し込んだ後、IMEを再開します。クリップボードを一切汚さないため、もっとも安定しています。

### MacOS：クリップボード経由＋履歴復元方式
ペーストの確実性を担保するためクリップボードを経由しますが、本来のコピペ履歴を破壊しないよう `copyQ` で履歴を退避・復元します。

```text
[ユーザーが固有名詞を入力]
       │
       ▼
1. copyQ の履歴記録を停止
2. pbcopy で入力文字列をクリップボードに強制セット
3. Cmd + V でペースト実行
4. copyQ を履歴記録を再開、最新の履歴をクリップボードに復元
```

### Linux：セカンダリ退避方式

Linux特有の「セカンダリ（Selection）」バッファを一時的な避難所として利用し、クリップボードの中身を保護します。

```text
[ユーザーが固有名詞を入力]
       │
       ▼
1. 現在のメインクリップボードを「セカンダリ」へ一時退避
2. 入力文字列をメインクリップボードにセットして Ctrl + V でペースト
3. セカンダリから元のデータをメインクリップボードへ復元
4. セカンダリをリセットして空にする
※注: Wayland環境など、ご利用のディスプレイサーバーの環境によってはクリップボードの復元が正常に動作しない場合があります。
```

### 便利な特殊操作
薙刀式のON/OFF: ゲームのプレイ時や、通常のQwerty配列に戻したい場合は、いつでも以下のショートカットでトグル切り替えが可能です。

Ctrl + Shift + 0 ── 薙刀式を一時停止（オフ）

Ctrl + Shift + 1 ── 薙刀式を再開（オン）
